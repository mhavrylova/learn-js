Теорія

Об'єкт може містити в собі властовості. Властивість - це пара ключ-значення, де ключ – це рядок (також називається “ім’я властивості”), а значення може бути будь-яким і будь-якого типу Методи створення об'єкту let user = new Object(); // синтаксис "конструктора об’єктів"
let user = {}; // синтаксис "літералу об’єкта"

Властивість має ключ (також відомий як “ім’я” або “ідентифікатор”) перед двокрапкою ":" і значення праворуч від нього. let user = { // об’єкт
name: "Іван", // за ключем "name" зберігаємо значення "Іван"
age: 30 // за ключем "age" зберігати значення "30"
}; Для звернення до властивостей використовується запис через крапку: user.name ('Іван'), user.age(30)

Щоб видалити властивість ми можемо використати оператор delete: delete user.age;

Ім’я властивості може складатися з декількох слів, але тоді воно має бути поміщено в лапки: "likes birds": true
Для властивостей, імена яких складаються з декількох слів, доступ до значення «через крапку» не працює, отримаємо синтаксичну помилку. Для таких випадків існує альтернативний спосіб доступу до властивостей через квадратні дужки. Такий спосіб спрацює з будь-яким ім’ям властивості: user["любить птахів"]
Квадратні дужки також дозволяють звернутися до властивості, ім’я якої може бути результатом виразу. Наприклад, ім’я властивості може зберігатися в змінній: let key = "любить птахів";
user[key] = true; Ми можемо використовувати квадратні дужки в літеральній нотації для створення обчислюваної властивості.
Квадратні дужки набагато потужніші за крапкову нотацію. Вони допускають будь-які назви властивостей та змінні. Але вони також більш громіздкі для написання.

ми можемо легко перевірити, чи існує властивість. Читання відсутньої властивості просто повертає undefined
Для цього також є спеціальний оператор "in". "key" in object поверне true, якщо в object є властивіть "key". Якщо ми опускаємо лапки, це означає, що ми вказуємо змінну, в якій знаходиться ім’я властивості.

Цикл "for...in"

for (key in object) {
// тіло циклу виконується для кожної властивості об’єкта
} // key - кожен ключ, object[key] - значення

Об'єкт, оголошений за допомогою const можна змінювати. const захищає тільки саму змінну від змін, зберігає тільки ссилку на об'єкт і її змінити не можна, але сам об'єкт міняти можна
