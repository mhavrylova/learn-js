Теорія

Function Declaration (Оголошення Функції)

function functionName() {
body; // крапка з комою тут
}

Function Declaration може бути викликана раніше (стадія ініціалізації), ніж вона оголошена. Тобто двіжок JS шукає всі Function Declaration і тсворює всі ці функції, а вже потім буде виконана решта коду.
Якщо Function Declaration знаходиться в блоці {...}, то зовні вона стає невидимою (блочна область видимості).

Function Expression (Функціональний вираз)

let functionName = function() {
body
}; // крапка з комою тут тут допускається віжсутність імені функції, адже сенс такого присвоювання - помістити функцію в змінну

Function Expression створюється, коли виконання доходить до неї і вже тоді може використовуватись.


Функція - це значення, яке являє собою дію і вона зберігається в змінній, якій присвоюється. З функією можна вчиняти як і з іншими значеннями, тобто навіть копіювати її в інші змінні
function sayHi() { // (1) створили
alert( "Привіт" );
}
let func = sayHi; // (2) скопіювали
func(); // Привіт // (3) викликали копію
sayHi(); // Привіт // початкова функція Якщо викликати ф-цію без фігурних дужок (alert functionName), то виведеться код цієї функції.

Функції колбеки

Колбеки - це функції, які передається як аргументи в виклику іншої функції і очікується, що колбек буде викликаний колись пізніше, якщо буде така необхідність

function ask(question, yes, no) {
if (confirm(question)) yes()
else no();
}

function showOk() {
alert( "Agree" );
}

function showCancel() {
alert( "Not Agree" );
}

// використання: функції showOk, showCancel передаються в якості аргументів ask
ask("Are you agree?", showOk, showCancel);
