Теорія

Рядки завершеного коду в JS слід розділяти знаком ;. В більшості випадків JS при новому рядку автоматично читає крапку-з-комою, навіть якщо вона там не стоїть. Але є моменти, коли ; не читається автоматично, наприклад перед квадратними дужками. Тому ; краще ставити самостійно

Часто в процесі написання коду потрібно писати коментарі, які не впливають на виконання коду. Однорядкові коментарі починаються з // і можуть займати як окремий рядок, так і йти після коди. Багаторядкові коментарі починаються з /* і закінчуються */

Якщо на проекті є застарілий код, який був написаний до ECMAScript 5 (ES5), то потрібно використовувати директиву "use strict" на початку сприпта. Це дозволить працювати старому коду в сучасному режимі.

Сучасний JS підтримує класи та модулі, які включають в себе "use strict" і якщо весь код погбудований на них, то "use strict" не потрібно додавати на початку коду
