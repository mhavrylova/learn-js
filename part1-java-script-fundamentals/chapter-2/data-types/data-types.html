Типи Даних

B JS значення відносяться до певного типу. Змінна може містити будь-які дані і вони можуть змінюватись (то була строка, тепер число). Мови програмування з такими можливостями називаються "динамічно типізованими".

B JS існує 8 типів даних:

  1. number
  2. bigint
  3. string
  4. boolean
  5. null
  6. undefined
  7. symbol
  8. object

Number / Число

Числом може бути як ціле значення, так і з плаваючою точкою. Для числового значення можливо примінити математичні операції.
Існують спеціальні числові значення - Infinity(безкінечність) и NaN (not a number). Також трапляються -0 та -infinity.
Infinity більше будь-якого числа. Його можна отримати діленням на нуль, чи задати явно.
NaN - це обчислювальна помилка і є результатом неправильної чи невизначеної математичної операції (наприклад, поділивши строку на число). Також будь-яка матиматична операція, в якій є NaN, поверне NaN (вийняток NaN ** 0 равно 1.
Математичні операції навіть якщо повертають помилку, не зупинять виконання коду.

BigInt

Для роботи з гігантськими числами, які більші ніж (253-1) або меньші ніж -(253-1) (тому що ці числа не можуть безпечно зберігатись в number. Щоб створити значення такого типу, в кінці числа ставлять n.

String / Строка

Строки поміщаються в кавички: 'string' або "string" або `string`.
' та " - прості і не мають різниці, але ` мають більше можливостей - дозволяють вирази та змінні вставляти в строку за допомогою ${…}: let name = "Marina";
// змінна
alert( `Hello, ${name}!` ); // Hello, Marina!
// Вираз
alert( `Persyk is ${4 * 2} years old` ); // Persyk is 8 years old

Boolean / Логічний тип

Булевий тип приймає лише значення true (правда) та false (брехня). Використовується для зберігання значення так або ні. Також можуть бути результатом порівнянь: let isGreater = 4 > 1;
alert( isGreater ); // true

Null

Спеціальне значення null являє собою "нічого", "пусто" або "значення невідомо" let age = null; // значення змінної age невідоме

Undefined

Спеціальне значення undefined являє собою "значення не було присвоєно", зазвичай використовується, щоб перевірити чи була призначена змінна let age;
alert(age); // undefined

Symbol

Використовується для створення унікальних ідентифікаторів в об'єктах.

“Символ” являє собою унікальний ідентифікатор і може бути ключем об'єкта. Створити символ можна за допомогою Symbol() Після створення символу ми можемо надати йому опис (так зване ім’я символу), в основному це корисно для процесу налагодження. Символи гарантовано будуть унікальними. Навіть якщо ми створюємо багато символів з однаковим описом, вони мають різні значення. Опис – це просто мітка, яка ні на що не впливає.

// Створюємо символ id з описом (іменем) "id" let id = Symbol("id");

Символи не перетворюються автоматично в рядок.
Символи дозволяють нам створювати “приховані” властивості об’єкта, до яких жодна інша частина програми не може випадково отримати доступ або перезаписати їх.
Наприклад, якщо ми працюємо з user об’єктами, які належать сторонньому коду. Ми хотіли б додати до них ідентифікатори.
Властивості, які є символами, не перебираються for..in циклом.

Object / Об'єкт

Тип об'єкт особливий, не примітивний як всі попередні (в примітивних зберігаються лиш прості дані). В об'єктах зберігають коллекції даних чи складні структури.

Оператор typeof

Оператор typeof повертає тип аргумента. Має 2 синтаксиса // Звичайний синтаксис
typeof 5 // "number"
// Синтаксис, як виклик функциії
typeof(5) // "number"

Щоб передати вираз, то треба поміщати його в дужки, тому що пріоритет typeof вищий за бінарні оператори: typeof 50 + " Квартир"; // "number Квартир" тому що спочатку відбувся typeof, а потім конкатенація
typeof (50 + " Квартир"); // "string" відбулась конкатенація, а тоді вже її typeof

typeof має деякі помилки, це склалось історічно ще від створення JavaScript: 1 - typeof null // "object", але це неправильно. null - спеціальне значення, що має окремий тип
2 - typeof alert // "function" - алерт це функція, але нема такого типу як функція, вона відноситься до об'єкту
